# Smart Internship Platform

Smart Internship Platform is a Laravel 13 and React 19 application for managing internships, student profiles, applications, match scores, and company applicant review workflows.

The project is currently implemented through Slice 7: Company Manages Applicants. The included Postman collection is organized by all slices. Requests for Slices 1-7 are sourced from the current application routes and frontend API modules. Later slices are included as dashboard data-source requests, smoke/security checks, or clearly marked planned endpoints where the current codebase does not yet expose a route.

## Tech Stack

- Backend: Laravel 13, PHP 8.3, Laravel Sanctum
- Frontend: React 19, React Router, Vite, Axios
- Database: SQLite by default for local development, MySQL/MariaDB supported through `.env`
- Tests: PHPUnit through `php artisan test`

## Requirements

Install these before running the project:

- PHP 8.3 or newer
- Composer
- Node.js and npm
- SQLite, MySQL, or MariaDB
- Git

On Laragon, PHP, Composer, MySQL, and a local web server are usually already available.

## Quick Start

From the project root:

```bash
composer install
npm install
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
npm run build
```

Run the backend and frontend during development:

```bash
php artisan serve
npm run dev
```

Open the frontend from the Vite URL printed by `npm run dev`, usually:

```text
http://127.0.0.1:5173
```

The API is served by Laravel, usually:

```text
http://127.0.0.1:8000/api/v1
```

## Environment Setup

Create `.env` from `.env.example`:

```bash
cp .env.example .env
```

For SQLite local development, use:

```env
DB_CONNECTION=sqlite
DB_DATABASE=database/database.sqlite
```

Then create the SQLite file if it does not exist:

```bash
php -r "file_exists('database/database.sqlite') || touch('database/database.sqlite');"
```

For MySQL or MariaDB, update:

```env
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=smart_internship_platform
DB_USERNAME=root
DB_PASSWORD=
```

After changing database settings:

```bash
php artisan config:clear
php artisan migrate --seed
```

## Seeded Data

Run:

```bash
php artisan migrate:fresh --seed
```

The seeder creates:

- An admin user:
  - Email: `admin@example.com`
  - Password: `password`
- Skills
- Student profiles
- Company profiles
- Internships
- Applications

Company and student seed users are generated by factories with random emails. For predictable Postman testing, use the Postman register requests to create known student and company users.

## API Base URL

All versioned API routes use:

```text
/api/v1
```

For local Postman usage:

```text
{{base_url}} = http://127.0.0.1:8000
{{api_url}} = http://127.0.0.1:8000/api/v1
```

Authenticated routes use Laravel Sanctum bearer tokens:

```http
Authorization: Bearer <token>
Accept: application/json
```

## Postman API Documentation

Import these two files into Postman:

- `docs/postman/Smart Internship Platform.postman_collection.json`
- `docs/postman/Smart Internship Platform Local.postman_environment.json`

Recommended run order:

1. Start Laravel with `php artisan serve`.
2. Select the `Smart Internship Platform Local` environment.
3. Run `Slice 1 - Authentication` register/login requests to create and store `student_token`, `company_token`, and `admin_token`.
4. Run `Slice 2 - Browse Internships` and `Slice 4 - Student Profile` to populate `skill_id` and `internship_id`.
5. Run `Slice 3 - Company Internship Management` to create a company-owned internship, test archive behavior, and create a fresh open internship for application testing.
6. Run `Slice 5 - Student Applications` to apply and store `application_id`.
7. Run `Slice 7 - Company Applicants` to list applicants and update application status.

The collection includes all slices:

- Slices 1-7: implemented API requests from the current route list.
- Slice 8: dashboard data-source requests built from existing student/company APIs.
- Slice 9: planned admin API requests, marked in the request descriptions because the current codebase does not yet expose those routes.
- Slices 10-11: polish, smoke, and hardening checks over current APIs.

## Deployment Notes

For a simple production deployment:

```bash
composer install --no-dev --optimize-autoloader
npm ci
npm run build
php artisan config:cache
php artisan route:cache
php artisan view:cache
php artisan migrate --force
php artisan storage:link
```

Set production environment values:

```env
APP_ENV=production
APP_DEBUG=false
APP_URL=https://your-domain.example
CACHE_STORE=database
QUEUE_CONNECTION=database
SESSION_DRIVER=database
```

Recommended server setup:

- Point the web server document root to `public/`.
- Run queue workers if background jobs are added later:

```bash
php artisan queue:work
```

- Schedule Laravel tasks if scheduled commands are added:

```bash
php artisan schedule:run
```

Optional Docker outline:

- PHP-FPM container for Laravel
- Nginx container serving `public/`
- MySQL or MariaDB container
- Node build step for Vite assets
- Volume or object storage for uploaded CV files

## Troubleshooting

If Postman receives `401 Unauthorized`, log in again and make sure the correct environment token is selected.

If a company route returns `403 Access denied`, confirm you are using `company_token`, not `student_token`.

If a student route returns `403 Access denied`, confirm you are using `student_token`.

If a route says a method is not supported, compare the URL against the route list:

```bash
php artisan route:list --path=api
```

For company applicant status updates, the correct route is:

```text
PATCH /api/v1/company/applications/{application}/status
```

not:

```text
PATCH /api/v1/applications/{application}/status
```
